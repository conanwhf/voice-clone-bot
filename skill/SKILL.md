# Voice Clone 对外技能接口

## 何时使用
- 用户显式指令你需要发一段语音、进入“语音模式”或是让你克隆某个特定的声音回复。
- 在语音交互场景下，每当你基于 OpenClaw 的核心生成了新的回复文本时。

## 底层通讯方式
这只是一个衔接 OpenClaw 和本地大容量语音合成模型的桥梁脚本。
`python scripts/tts_client.py --text "你好，世界" --ref_audio "/用户/所在/的绝对音频路径.ogg"`

脚本成功时会反馈 `MEDIA:/生成/录音/的路径.ogg`，你可以用其组装最终答复，实现通过 Telegram 发语音！

## 默认接管流程

如果你被明确要求了处于语音模式，针对接下来的交谈，请遵守以下严密逻辑：

1. **查找语料参考 (ref_audio)**
   - 检查用户有没有指定特定的媒体录音作参考？如果有，把最近收到保存在 `download/media/` 下的那个音频提取出来做 `--ref_audio` 的参数（例如：`/path/to/whisper_saved.ogg`）。
   - 如果用户没有指定特殊的声音源，你可以留空该参数（脚本将使用缺省的预设克隆对象）。
2. **组织想要说的文本 (text)**
   - 对于文本合成极高拟真度，请不要发送过长的裹脚布段落（会导致显存不足导致延迟），尽最大可能把语句精炼在百字以内（如果非常长请自觉思考为分段答复）。
3. **发起运行生成指令**
   你需调用该脚本（使用绝对路径调用）：
   ```bash
   python3 /Volumes/External/OpenClawWork/share/project/VoiceCloneBot/openclaw-voice-clone/skill/scripts/tts_client.py --text "你精心准备的简短答复句" --ref_audio "提取的用于音色锚点的绝对路径（需存在）"
   ```
4. **获取回执并在当前对话抛出**
   如果一切顺利，脚本的控制台将返回仅有一行的绝对引用路径！
   你在响应客户端时，仅需遵守以下强制规定来发声：
   **单起最后一行添加（不要包含其它废话或空格）：**
   `MEDIA:<绝对被生成的音频路径地址>`
